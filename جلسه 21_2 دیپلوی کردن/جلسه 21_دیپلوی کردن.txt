این جلسه: دیپلوی کردن.
اول از همه میبریمش پروژه رو رو گیت هاب.
ساخت اکانت هیروکو
نصب هیروکو سی ال آی.
اگه توی ترمینال نمیشناختش بریم توی اینوایرنمنت وریبل اضافه اش کنیم توی قسمت پس.
C:\Program Files\heroku\bin
البته جایی که نصب شده شاید جای دیگه نصب باشه. خلاصه پوشه bin ش رو باید بذاریم.
اگه شناخت که حله.
برای تشخیص شناخت تو ترمینال بنویسم.
heroku login
بعد با بروزر با اکانتم لاگین کنم.
بعد توی ترمینال مینویسه که لاگین شدی.

البته به خاطر تحریم و عدم همخوانی با فیلتر شکن و دی ان اس شکن و خلاصه هر چی بود نتونستم از تو ترمینال لاگین کنم.
بقیه مراحل رو مینویسم که دیگه فیلم رو نبینم و اگه یه روز تونستم لاگین کنم، این مراحل رو انجام بدم.


1-heroku login
و بعد لاگین کنم.
2-pip install gunicorn که انجام دادم. برای دیپلوی کردن باید نصب باشه رو سیستم.
3-یه فایل به نام زیر درست میکنیم که کنار manage.py توی اولین صفحه اصلی پروژه. این فایل فقط برای هیروکو هست و نه سایت های دیگه. اسمش این باشه.
Procfile
4- عبارت زیر رو داخلش مینویسیم و فایل رو ذخیره میکنیم و میبندیم.
web: gunicorn config.wsgi --log-file -
5-داخل ستینگز اصلی پروژه ALLOWED_HOSTS رو  داخل لیستش استرینگ * رو اضافه میکنیم.
6-روی گیتهاب آپدیت پوش میکنیم تغییرات رو.
7- دستور زیر رو میزنیم.
heroku create
یه اپی میسازه برای ما تو سایت خودشون.
8- دستور زیر:
heroku config:set DISABLE_COLLECTSTATIC=1
رو میزنیم و منتظر میمونیم درخواست رو بفرسته و جواب بده.
9-اگه اوکی بود دستور زیر رو بزنیم.
git push heoku main
که main اسم برنچمون بود.
که در واقع گفتیم بفرستش به هیروکو.
یه مدت طول میکشه تا جنگو و پکیج های لازم که تو ریوکایرمنتز گذاشته بودیم رو نصب کنه و کاراش رو بکنه.
آخرش یه آدرس به ما میده که با اسم همون اپی که خودش ساخته هست و میتونیم بریم داخلش.
10- حالا میتونیم به هیروکو بگیم که رو سرور خودشون چیکار کنه. مثلا
heroku run python manage.py migrate
heroku run python manage.py createsuperuser
و غیره.
11- دستور زیر
heroku open
بعدش واردش میشیم و تامام.
